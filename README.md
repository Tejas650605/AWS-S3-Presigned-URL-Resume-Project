# ☁ AWS S3 Secure Resume Sharing Project

## 📌 Project Overview
This project demonstrates secure resume sharing using Amazon S3 Presigned URLs in AWS Cloud.

The S3 bucket and uploaded resume object are kept completely private while temporary access is securely provided using a presigned URL valid for only 2 hours.

This project focuses on AWS cloud security, temporary access management, and secure file sharing techniques.

---

# 🚀 Key Features

✅ Private S3 Bucket Configuration  
✅ Private Resume Object Access  
✅ Static Website Hosting Disabled  
✅ Secure Presigned URL Generation  
✅ Temporary Access Expiration (2 Hours)  
✅ Cloud Security Best Practices  
✅ Secure Resume Sharing Without Public Access  

---

# 🛠 AWS Services Used

| Service | Purpose |
|---|---|
| Amazon S3 | Resume Storage |

# 📖 Implementation Steps

## 🔹 Step 1 — Create S3 Bucket
- Open AWS S3 Console
- Create a new bucket
- Enable Block Public Access
- Keep bucket private
- Disable static website hosting

---

## 🔹 Step 2 — Upload Resume
- Upload resume PDF file
- Keep object permissions private
- Verify secure object storage

---

## 🔹 Step 3 — Generate Presigned URL
- Select uploaded resume object
- Open object actions
- Generate presigned URL
- Set expiration time to 2 hours

---

## 🔹 Step 4 — Verify Secure Access
- Open generated URL
- Access resume temporarily
- Verify automatic expiration after 2 hours

---
